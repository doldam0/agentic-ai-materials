# Agentic AI 개요

Agentic AI는 자율적으로 목표를 달성하기 위해 행동하는 AI 시스템입니다.
기존 LLM이 단순히 질문에 응답하는 것과 달리, Agentic AI는 계획을 수립하고, 도구를 사용하며, 자신의 결과를 평가하고 개선합니다.

## 핵심 구성요소

### 1. Planning (계획)
복잡한 작업을 작은 하위 작업으로 분해하는 능력입니다.
- Task Decomposition: 큰 작업을 실행 가능한 단위로 분해
- Subgoal Setting: 중간 목표 설정으로 진행 상황 추적
- Ordering: 작업 간 의존성 고려하여 실행 순서 결정

### 2. Tool Use (도구 사용)
외부 도구와 API를 호출하여 기능을 확장합니다.
- 웹 검색, 데이터베이스 쿼리
- 수학 계산, 코드 실행
- 파일 읽기/쓰기, API 호출

### 3. Memory (메모리)
대화 컨텍스트와 과거 경험을 기억합니다.
- 단기 메모리: 현재 대화의 컨텍스트
- 장기 메모리: 사용자 선호도, 과거 작업 결과

### 4. Self-Reflection (자기 평가)
자신의 출력을 평가하고 개선하는 능력입니다.
- 답변의 정확성 검증
- 논리적 일관성 확인
- 품질 기준 충족 여부 확인

## LangChain 프레임워크

LangChain은 LLM 기반 애플리케이션 개발을 위한 오픈소스 프레임워크입니다.

주요 특징:
- 컴포넌트를 체인(Chain) 형태로 연결
- 에이전트, 메모리, 툴 추상화 제공
- 다양한 LLM 벤더 지원 (OpenAI, Anthropic, Google 등)

## RAG (Retrieval-Augmented Generation)

RAG는 검색(Retrieval)과 생성(Generation)을 결합한 기술입니다.

장점:
- 환각(Hallucination) 감소: 실제 문서 기반 답변
- 최신 정보 활용: 학습 데이터 이후 정보도 사용 가능
- 도메인 특화: 사내 문서, 전문 지식 활용

파이프라인:
1. 사용자 질문 입력
2. 벡터 저장소에서 관련 문서 검색
3. 검색된 컨텍스트와 함께 LLM에 전달
4. LLM이 컨텍스트 기반 답변 생성

## 주요 도시 기온 데이터

- 서울: 평균 기온 12°C
- 부산: 평균 기온 15°C
- 제주: 평균 기온 18°C

참고: 섭씨를 화씨로 변환하려면 (°C × 9/5) + 32 공식을 사용합니다.
